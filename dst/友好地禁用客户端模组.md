# 自己创建一个【友好地禁用客户端模组】

## 前言
创意工坊中有这样一个模组：[友好地禁用客户端模组](https://steamcommunity.com/sharedfiles/filedetails/?id=2899958182)，它能自动禁用玩家的客户端模组，并且在退出世界后恢复，不需要玩家执行任何手动操作。

不过它实际上需要你订阅模组[游戏内主菜单[客户端]](https://steamcommunity.com/sharedfiles/filedetails/?id=2893492379)，该模组对于其中的很多功能缺乏开关配置项，如果你不想要这些功能，可以尝试自己创建一个【友好地禁用客户端模组】模组。

## 原理

原理与[友好地禁用客户端模组](https://steamcommunity.com/sharedfiles/filedetails/?id=2899958182)相同：在存档的 `modoverrides.lua` 中启用 `client_mods_disabled` 配置项。

`modoverrides.lua` 的结构是这样的：

```lua
return {
  ["workshop-1"] = {
    configuration_options = {},
    enabled = true 
  },
}
```

模仿这一结构，我们只需要在 `modoverrides.lua` 中增加以下代码：

```lua
return {
  -- 以下为添加内容，禁用客户端模组
  client_mods_disabled = {
    configuration_options = {},
    enabled = true
  }
  -- 添加内容结束
}
```

但是每次重启世界，模组配置文件都会重新写入，导致我们新增的代码丢失。因此我们必须将它创建为一个服务器模组，让世界启动时自动载入这一配置。

## 实现

本文提供了一个[懒人包](https://pan.baidu.com/s/1UcR9cZcfH7Tp1npC772_VA?pwd=qxsr)，你可以直接下载解压到步骤 1 的文件夹路径下，然后执行步骤 3 即可。

### 自创建流程

1. 在 `SteamLibrary\steamapps\common\Don't Starve Together\mods` 路径下新建文件夹，命名为 `client_mods_disabled`。
2. 向文件夹中写入 `modinfo.lua` 文件，内容如下：
   ```lua
   name = "友好地禁用客户端模组"
   description = "通过往 modoverrides.lua 写入 client_mods_disabled 配置来实现友好地禁用客户端模组。"
   author = "Meltryllis"
   version = "0.1"
   -- icon_atlas = "modicon.xml"
   -- icon = "modicon.tex"
   dst_compatible = true
   all_clients_require_mod = true
   api_version = 10
   configuration_options = {}
   ```

3. 然后在服务器模组中启用本模组即可。

> 注意：将本模组上传至创意工坊是无效的，因为创意工坊模组会被赋予固定格式的名称—— `["workshop-工坊ID"]`。
